## RC Image Download Deep Dive (11:33 - 11:41 AM)
- **Exhaustive testing of all download approaches:**
  - Bearer token to `dl.mvp.ringcentral.com/file/{id}` → 404
  - access_token query param → 404
  - Platform API `/restapi/v1.0/glip/files/{id}` and `/content` → 404
  - `/team-messaging/v1/files/{id}` → 404
  - `app.ringcentral.com/files/{id}` → 200 but returns HTML (web app page, not file)
  - Tried curl with verbose, various header combos, redirect following, MVP subdomains
  - All 404s from CloudFront CDN: "Resource for parameter [fileId] is not found"
- **Conclusion:** `contentUri` from RC API (`dl.mvp.ringcentral.com`) NOT accessible via JWT/API tokens. Likely requires web session auth.
- **Current state:** Attachment metadata (name, type) IS visible to agent. File content not downloadable.
- **RC docs confirm:** "The access_token query parameter is only supported on media.ringcentral.com. It is not supported for Glip." Team messaging files on `dl.mvp.ringcentral.com` are NOT accessible via API tokens. Only the web app (with session cookies) can download them. This is a documented RC platform limitation.
- **Dave provided URL pattern:** `dl.mvp.ringcentral.com/company/{companyId}/file/{fileId}?stored_file_id={storedFileId}` but `stored_file_id` and `companyId` (20111835137) are not available through the public API, and even with the full URL, API Bearer tokens get 404.
- **SOLVED via browser cookies!** Dave logged into app.ringcentral.com in openclaw browser. Extracted `jtk` cookie from `dl.mvp.ringcentral.com` via CDP. Cookie is a JWT with 60-day expiry (expires 2026-05-01). Download works with just the cookie on the simple contentUri (no company/stored_file_id needed).
- **Implementation:** `getMvpCookie()` in api.ts connects to CDP at `127.0.0.1:18800`, extracts cookies, caches them with expiry. `downloadFile()` uses cookie instead of Bearer token. Files saved to `~/.openclaw/workspace/media/rc/`.
- **Requirement:** openclaw browser must be logged into app.ringcentral.com once to seed cookies. Cookie cached to file (`media/rc/.cookie-cache.json`), valid for 60 days. Browser doesn't need to stay open after seeding.
- **CONFIRMED WORKING at 12:44 PM:** Full pipeline: image sent in RC → plugin detects attachment → loads cookie from file cache → downloads to workspace → agent sees and describes image content. ✅
